乐玩插件接口说明 v7.53

ReadUniversalEx


函数简介:

读取指定地址的数据

 

 

函数原型:

string  ReadUniversalEx(addr,type,len)

 

 

参数定义:
 

addr 字符串: 用字符串来描述地址，等同于OD的地址描述，数值必须是16进制,里面可以用[ ] + -这些符号来描述一个地址。+表示地址加，-表示地址减,*表示地址乘
         例如:

1.         "4DA678" 最简单的方式，用十六进制绝对数值来表示地址

2.         "[4DA678]+3A" 用绝对数值加偏移，相当于一级指针

3.         "[[DA678]+3A]+5B]+8" 这个是一个三级指针

4.         "[[DA678]+3A]+5B+1*4]+8" 这个是一个带乘法的三级指针

5.         "[<360SE.exe>+DA678]+3A" 用模块定基址的方式，也是一级指针

总之熟悉OD的人 应该对这个地址描述都很熟悉,我就不多举例了

type 整数型:要读取的类型，默认为0,取值方式如下：

                  0：整数型（int）

                  1：短整数型（short ）

                  2:  长整数型（long ）

                  3:  字节型（byte）

                  4:  单精度浮点（float）

                  5:  双精度浮点（double）

                  6:  GBK字符串

                  7:  Unicode字符串

                  8: 字节数组（Array of Bytes 以十六进制形式返回）

                  9: 字节数组（Array of Bytes 以十进制形式返回）

len  整数型: 欲读取长度，读取的类型小于等于5时，此参数无效，6或者7时，如果此参数为0时，自动判断文本的长度，否则读取指定长度，8或者9时，此参数是必须的。默认为0。

返回值:

字符串：

成功返回读取到的数值（以字符串形式返回）

 

示例:

value = lw.ReadUniversalEx(  "[[DA678]+3A]+5B+1*4]+8" )

MessageBox  value


提示：此命令支持对64位进程的读写，建议使用。

注意,在读写之前一定要保证该对象使用过 SetMemoryProcess命令设置过读取的目标进程.参见(SetMemoryProcess.html)之后的所有内存相关命令都一样,不再提示!
